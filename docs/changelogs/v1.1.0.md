# v1.1.0 — Process And Agent Design Layer

**Date:** 2026-04-10

This release adds a new planning layer above direct skill execution.

`cross-plat-skills` started as a cross-platform skill library. In `v1.1.0`, it also becomes a process-aware system: complex work can now be decomposed into reusable process entries, routed through agent-structure design when needed, validated before execution, and written back into the process registry after execution.

---

## What's New

### New Agent And Process Design Skills

- `process-decomposer`
  checks the process registry, classifies complexity, and writes reusable process entries
- `agent-architect`
  designs single-agent or multi-agent execution structures for decomposed processes
- `setup-evaluation`
  validates decomposition and architecture before execution
- `skill-finder`
  checks whether a capability is already covered by an existing skill
- `tool-finder`
  checks tool availability and setup requirements
- `create-agent-prompt`
  writes focused role prompts for agents in multi-agent topologies

### Expanded Orchestration

`project-orchestrator` now sits explicitly downstream of the new process layer. It remains the execution wrapper and the feedback owner for process-backed runs.

### Reusable Process Memory

The repo now includes `docs/processes/process.md` as the initial process registry. The intended flow is:

user request
→ process lookup and triage
→ decomposition
→ architecture design when needed
→ setup validation
→ execution
→ execution feedback written back to the process entry

### Current-State Architecture Doc

The repo now includes `docs/architecture.md`, which documents the architecture that is actually present in the checkout today.

---

## Why This Matters

Most agent workflows are still assembled by hand for each new business process. This release moves the repo one step toward a more reusable model:

- process reasoning happens before execution
- process artifacts are reusable
- architecture decisions are separated from execution
- complex setups get a validation step before they run

This is still a lightweight, portable skill system rather than a full runtime. That remains intentional.
