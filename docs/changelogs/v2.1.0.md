## Summary

**Agents now remember, and the library now polices itself.** Since v2.0.0 made *skill creation* disciplined, v2.1.0 makes *every other moment of agent work* disciplined: a 10-skill memory suite for cross-session continuity, auto-triggered memory checkpoints at every producer event, a symmetric Session Lifecycle (Session Start + Session End) in AGENTS.md, a retroactive bootstrap path for existing codebases, a chat-learnings ↔ improve-skills feedback loop so in-session discoveries can't go missing, and three new self-enforcing structural flags in `validate-skills` so silent library rot becomes visible.

**Upgrade now if** you ship long-running agent work across sessions, onboard agents to existing codebases, or want your skill library to stay clean without manual policing.

**Try it:**
```bash
git pull
bash ~/.agent-loom/install.sh --update      # macOS / Linux
.\install.ps1 -Update                       # Windows
```

Then ask any AI tool: *"what happened last time"* (memory-startup), *"backfill agent infra for this existing repo"* (retroactive-project-setup), *"remember this for the skill"* (learn-from-chat).

## Added — Memory Suite (10 skills) — *Stop losing context between sessions*

A bounded, agent-authored memory layer that survives session resets, agent swaps, and tool switches. Every skill in the suite has a single narrow job and a strict size budget. **Who benefits:** anyone whose AI agent currently re-reads the whole repo every session because the previous session's context evaporated. **Why it matters:** "what did we decide last time" is the highest-frequency lost-context question in agentic work. **Why install now:** every session you start cold today is a session you could resume warm tomorrow.

- **[`memory`](.agents/skills/memory/)** — Orchestrator. Routes startup, recall, capture, handoff, decisions, promotion, compaction, audit, and forgetting to the right sub-skill. Security gate fires at Step 1 before any external content enters memory.
- **[`memory-startup`](.agents/skills/memory-startup/)** — Loads **bounded** working context: `docs/memory/project-index.md`, latest handoff entry, and only directly relevant decisions. Never reads every memory file.
- **[`memory-capture`](.agents/skills/memory-capture/)** — Saves durable project memory; rejects trivial, sensitive, duplicate, or obvious content. Used by every producer-event auto-trigger.
- **[`memory-handoff`](.agents/skills/memory-handoff/)** — Writes compact next-agent handoffs. Fires automatically at session end and before agent/tool switches.
- **[`memory-decision`](.agents/skills/memory-decision/)** — Records decisions with rationale, alternatives, assumptions, status, and concrete revisit triggers. Distinguishes deferred from rejected.
- **[`memory-recall`](.agents/skills/memory-recall/)** — Task-relevant retrieval. Pulls only the rows that match the current task — no full-store loads.
- **[`memory-promote`](.agents/skills/memory-promote/)** — Gates local-to-global memory promotion into `~/.agent-loom/memories/`. Global memory is a curated operating manual, not a journal.
- **[`memory-compact`](.agents/skills/memory-compact/)** — Enforces compaction before any over-budget write. Strict global memory size budgets.
- **[`memory-audit`](.agents/skills/memory-audit/)** — Audits memory health, stale entries, contradictions, broken routing, bloat. Cross-file overlap check catches `Confidence: low` decisions that also appear in `current-state.md` Active Risks.
- **[`memory-forget`](.agents/skills/memory-forget/)** — Deletes, redacts, archives, or retires memory.

## Added — Retroactive Project Setup — *Onboard agents to an existing codebase without touching a single line of source code*

- **[`retroactive-project-setup`](.agents/skills/retroactive-project-setup/)** (Atomic, 188 lines) — Bootstraps the full agent layer (AGENTS.md + `docs/architecture.md` + `docs/product-soul.md` + `docs/adr/ADR-0001-initial-backfill.md` + `docs/memory/` seed) over a project that already has code but no agent infrastructure. **Strict write-allowlist** enforced in workflow Step 1 — refuses to modify source code, configs, or anything outside the allowlist. Surveys repo manifests, README, CHANGELOG, source samples, and the last 50 commits to infer everything answerable; runs a ≤6-question targeted interview only for genuine gaps. Tags low-confidence inferences as `[INFERRED — confirm]` so future agents don't mistake reconstruction for live capture. Refuses on populated AGENTS.md, routing to `project-setup UPDATE_ONLY=true` instead. **Who benefits:** anyone with a real production codebase who wants agent support without a rewrite. **Why it matters:** the alternative is either no agent infra (agents flail) or a fabricated one (agents follow imaginary rules).

## Added — Workflow & Library Infrastructure

### Mandatory Auto-Trigger Memory Checkpoints

A canonical event-to-sub-skill registry in `memory/SKILL.md` makes memory work auto-fire at every producer event — not only when the user says "remember this." Trigger map: changelog → `memory-capture`, ADR → `memory-decision`, spec/plan/PRD → `memory-capture`, skill creation → `memory-capture`, major commit (>20 files or breaking) → `memory-capture`, session end → `memory-handoff`. **6 producer skills** (`generate-changelog`, `architectural-decision-log`, `feature-spec`, `implementation-plan`, `prd-writing`, `universal-skill-creator`) each gained a final "Memory Checkpoint (Mandatory)" step pointing at the registry. **AGENTS.md** gained a top-level "Memory Checkpoints — Mandatory" section at the same precedence as "Startup Skill Loading."

### Session Lifecycle — Mandatory (Session Start added)

AGENTS.md previously instructed end-of-session memory checkpoints but said nothing about session start. New **Session Lifecycle — Mandatory** section makes the contract symmetric:
- **Session Start** — invoke `memory-startup` (bounded, not the full store), run `git status` + `git log --oneline -5`, state recovered context in 2–4 lines, wait for user confirm. Opt-out: user says "fresh start" or "ignore prior context".
- **During & End** — preserved producer-event auto-trigger guidance.

The block is now in the `project-setup` template so **every new project's AGENTS.md inherits the discipline** — not just agent-loom itself.

### Chat-Learnings ↔ `improve-skills` Feedback Loop

`learn-from-chat` and `improve-skills` are now two ends of one closed loop:
- **New `improve-skills TARGETED` mode.** Invoked as `improve-skills TARGET=<skill> [SKIP_RESEARCH=true]`. The canonical entry point for any single-skill structural fix. `SKIP_RESEARCH=true` skips only Step 2e (research) when the change source is already trusted; never bypasses security, validate, deconflict, size, or close steps.
- **Step 1b Chat-Learnings Ingest.** `improve-skills` now reads `docs/learnings/chat-learnings.md` at the start of every pass, triages each `Status: OPEN` entry into one of four verdicts (`IMPLEMENTED (pre-existing)`, `REJECTED`, `DEFERRED`, or keep `OPEN` for Step 2g merge), and writes terminal statuses back in new Step 2l.
- **`learn-from-chat` Step 5 escalation gate.** Append-only edits (one bullet to Gotchas or Hard Rules, one-line workflow phrasing fix, citation) stay in-scope. Anything restructure-class (new/renumbered workflow step, section restructure, new `references/` file, routing-trigger change, 200-line gate crossing) MUST escalate to `improve-skills TARGET=<skill> SKIP_RESEARCH=true`.
- **Mandatory `Status` field** on every `chat-learnings.md` entry. Values: `OPEN`, `IMPLEMENTED ([date], <skill> v<ver>)`, `ESCALATED (improve-skills TARGET=<skill>, [date])`, `REJECTED (<reason>)`, `DEFERRED (<reason>)`. A blank Status hides the learning from the next improvement pass — the rule is now visible.

### `architectural-decision-log` v1.1 — `SYNTHESIS=true` Mode

New named mode for retrospective ADR backfill from observed repo state, parallel to `project-setup`'s `RETROACTIVE=true`. Skips Step 2's interview (no live forces are available), reads Context from README + manifests + commit history, lists at least 2 Alternatives marked `[INFERRED]`, derives Consequences from observed code shape, writes `Status: Accepted (retrospective)`, and MUST append "Decisions inferred from repo state as of YYYY-MM-DD; not contemporaneous" to Context. New Gotcha makes the inference-tagging requirement non-negotiable ("`SYNTHESIS` mode is honest, not confident — removing the tags to make the ADR look cleaner turns the file into a confabulation hazard"). Invoked by name from `retroactive-project-setup` Step 5.3.

### `validate-skills` v1.1 — Two New Structural Flags

- **Loader-Safety Check (Step 2a + LOADER SAFETY (P0) report block).** Catches BOM, byte-0 not `-`, missing closing `---` fence, and (in the rubric) >1024-char descriptions. Any failure is P0 — the skill will not load on at least one major agent platform. Verified clean across all 90 current skills.
- **Producer-Skill Checkpoint Audit (Step 4c).** Reads `memory/SKILL.md` registry once, classifies every skill as a "producer" by its output directory / description / registry-trigger entry, then greps for an invocation of the matching memory sub-skill. Missing → structural flag "Missing memory-checkpoint registration." Mirrored row in `references/validation-rubric.md`. **Makes the checkpoint rule self-enforcing instead of a written prayer.** All 6 current producer skills pass.

### `universal-skill-creator` v2.2 → v2.3 — Step 11 Library Sync (Mandatory)

The "Step 8 auto-chain" library-sync was an AGENTS.md invariant but not a numbered step in the creator's workflow — easy to skip selectively. Now an **explicit, numbered, mandatory Step 11** invokes `library-skill` with trigger `new skill added — <skill-name>` after Step 10 (cross-link repair). Verification checklist gained a matching item.

## Compatibility / Migration

- **All existing skills, AGENTS.md files, install scripts, and integrations continue to work.** Memory infrastructure is opt-in — old projects without `docs/memory/` and without the Session Lifecycle block keep working; new projects get both by default via `project-setup`.
- **Producer skills you've authored externally** should add a final "Memory Checkpoint" step pointing at `memory/SKILL.md` to pass the new `validate-skills` Step 4c flag. Without it, `validate-skills` will surface a structural flag — non-blocking but visible in every audit.
- **`improve-skills` invocations** — old `improve-skills` calls (FULL_PASS mode, no `TARGET=`) work unchanged. The new `TARGET=<skill> [SKIP_RESEARCH=true]` mode is purely additive.
- **`architectural-decision-log` invocations** — `INTERACTIVE` mode is the default; old calls behave identically. `SYNTHESIS=true` is opt-in.
- **`docs/learnings/chat-learnings.md`** — old entries without a `Status` field are treated as `OPEN` by `improve-skills` Step 1b. No backfill required; the next improvement pass triages them.
- **`agentskills validate` CLI remains unavailable** in this environment; structural and loader-safety checks were performed manually. Next agent should rerun the full `validate-skills` sweep in a clean environment to confirm the new Step 2a and Step 4c flags behave correctly across the whole library, not just the spot-checks performed during development.
