# Canvas Selector

Load in `business-modeling` Step 2. Pick exactly one primary canvas. Optional VPC appendix only when value-prop fit is the dominant uncertainty.

## Decision Tree

```
Q1: Is the primary uncertainty Problem / Solution / WTP / Channel?
├── Yes → Lean Canvas (default)
└── No, it's…
    ├── Revenue / Cost / Partners / Operations → Business Model Canvas (BMC)
    └── Whether segment Pains/Gains map to Product → Value Proposition Canvas (VPC)
```

## Stage Heuristic

| Stage | Default canvas | Reason |
|---|---|---|
| Pre-launch (no users) | Lean Canvas | Problem and solution are still hypotheses |
| Pre-revenue (some users, no payment) | Lean Canvas + VPC appendix | Need to confirm WTP and value-prop fit |
| Pre-PMF (paying users, unclear retention) | Lean Canvas → BMC | Map operations as users multiply |
| Post-PMF, scaling | BMC | Optimisation of operations / partnerships |
| Pivot of mature business | BMC | New activities / resources / partnerships needed |
| Adding new segment to existing product | VPC | Segment-specific fit is the question |

## Canvas Quick Reference

### Lean Canvas (Maurya, 2010)
9 boxes, 1 page, optimised for early-stage uncertainty. Replaces BMC's `Customer Relationships`, `Key Activities`, `Key Resources`, `Key Partners` with `Problem`, `Solution`, `Key Metrics`, `Unfair Advantage`.

Boxes:
1. **Problem** — top 3 pains for the segment
2. **Customer Segments** — specific persona(s); name early adopters
3. **Unique Value Proposition** — single clear, compelling message
4. **Solution** — top 3 features
5. **Channels** — path to customers (one wedge)
6. **Revenue Streams** — model + pricing
7. **Cost Structure** — fixed + variable
8. **Key Metrics** — actionable numbers that prove progress
9. **Unfair Advantage** — what cannot easily be copied/bought

### Business Model Canvas (Osterwalder, 2010)
9 boxes, 1 page, optimised for operational thinking.

Boxes:
1. **Customer Segments**
2. **Value Propositions**
3. **Channels**
4. **Customer Relationships** (acquisition / retention / upsell mechanics)
5. **Revenue Streams**
6. **Key Resources** (physical / IP / human / financial)
7. **Key Activities** (production / problem-solving / platform)
8. **Key Partnerships** (suppliers / strategic alliances / coopetition)
9. **Cost Structure** (cost-driven / value-driven; fixed / variable)

### Value Proposition Canvas (Strategyzer)
2 sides; zooms into ONE segment + ONE value proposition.

Customer Profile (right side):
- **Customer Jobs** — functional, social, emotional jobs the user is trying to get done
- **Pains** — bad outcomes, obstacles, risks before/during/after the job
- **Gains** — required, expected, desired, unexpected positive outcomes

Value Map (left side):
- **Products & Services** — what you offer
- **Pain Relievers** — how you eliminate/reduce pains
- **Gain Creators** — how you produce gains

Fit when: products match jobs, relievers match pains, creators match gains. Anything unmatched on either side is a flag.

## When to use TWO canvases

Default is ONE. Allowed exceptions:

- **Lean Canvas + VPC appendix** when value-prop-fit is the dominant uncertainty AND the founder needs to map operations.
- **Multiple Lean Canvases** when one venture is genuinely multi-sided (rare; usually the founder is conflating two ventures).

Never default to all three. The user picks ONE primary canvas. If they push back, ask which uncertainty is dominant and pick from the decision tree.
