# v1.3.0 ‚Äî Skill Deconfliction, Context-Aware Routing & Library Cleanup

**Date:** 2026-04-18

This release adds two new meta skills for skill library hygiene and intelligent routing, renames `agent-creator` to `agent-launcher` to eliminate naming confusion, fixes trigger overlaps and intent diversity across 9 skills, and extracts routing logic from `project-orchestrator` into a dedicated skill.

---

## Breaking Changes

- **Renamed `agent-creator` ‚Üí `agent-launcher`** ‚Äî update any references in custom workflows, architecture specs, or prompts. All internal cross-references already updated.

## Added

- **`skill-deconflict`** ‚Äî new meta skill that detects naming collisions (similar-sounding names for different skills), overlapping intent triggers (multiple skills matching the same user phrase), and insufficient intent diversity (too few or too similar trigger phrases). Returns PASS/RENAME/REVISE verdicts. Wired into `universal-skill-creator` (Step 8) and `improve-skills` (Step 2h) as a mandatory gate.

- **`skill-routing`** ‚Äî new meta skill that matches user requests to the right skill using trigger matching, project context, and conversation history. Scores ambiguity 1‚Äì10: silently resolves at 1‚Äì6 using context signals (project phase, conversation chain, trigger precision, upstream/downstream position); asks one binary disambiguation question at 7‚Äì10. Owns the full skill routing table (extracted from `project-orchestrator`).

## Changed

- **`universal-skill-creator`** ‚Äî added Step 8 (deconflict name and triggers via `skill-deconflict`) before validation. Steps renumbered 8‚Üí11.

- **`improve-skills`** ‚Äî added Step 2h (deconflict name and triggers via `skill-deconflict`) in the per-skill improvement cycle. Steps renumbered 2h‚Üí2k.

- **`project-orchestrator`** ‚Äî extracted the Skill Routing Table and inline disambiguation logic into `skill-routing`. Step 2 now invokes `skill-routing` and classifies the result. Reduced from 214 ‚Üí 168 lines.

- **`inversion`** ‚Äî removed trigger overlap with `pre-mortem` and `adversarial-hat` ("pre-mortem", "what could go wrong", "stress test this plan"). Added 4 new triggers: "reverse engineer the goal", "think about it backwards", "what's the opposite of success here", "how would we sabotage this".

- **`pre-mortem`** ‚Äî added 3 triggers for intent diversity: "risk analysis before launch", "what kills this project", "what could go wrong".
