**Version:** v2.0.0

## Summary

**Three new end-to-end workflows, a library that no longer drifts, and changelogs that read like a product.** Since v1.5.0 (Experimentation suite), agent-loom has added: a complete spec-driven-development workflow (constitution → spec → plan → cross-check), a complete pre-decision business-idea lifecycle (generate → model → evaluate → discover), a complete frontend-design workflow that fights vibecoded UI, plus a hard library-quality invariant that stops skills silently rotting. **Upgrade now if you ship features, ship products, or ship UI** — every workflow you actually use is now sharper.

**Try it:**
```bash
git pull
bash ~/.agent-loom/install.sh --update      # macOS / Linux
.\install.ps1 -Update                       # Windows
```

Then ask any AI tool: *"build a frontend for X"*, *"spec-driven development for Y"*, or *"is this a good business idea"*.

## Breaking Changes

🚨 **`agent-architect` → `agent-builder` (already shipped in v1.5.0 series; documented here for v2.0.0 cutover):** any custom workflow or prompt referencing `agent-architect` must be updated to `agent-builder`.

🚨 **`setup-evaluator` → `setup-evaluation` in skill cross-references:** the skill name is now used consistently in `agent-builder` and `project-orchestrator` routing. The agent name (`setup-evaluator`) is unchanged. Removes "skill not found" errors after agent-chain decompositions.

🚨 **Skill Creation Invariant (AGENTS.md):** agents are no longer permitted to write `.agents/skills/<name>/SKILL.md` directly — every skill creation must route through `universal-skill-creator` so the validate → deconflict → cross-link → library-update chain fires automatically. **Migration:** any external automation that wrote SKILL.md by hand must invoke the creator instead with the same brief. Human-authored commits are still allowed.

## Added — 3 New Skill Suites (14 skills)

### 🧱 Spec-Driven Development Suite (4 skills) — *Ship features your AI agents actually implement correctly*

A GitHub Spec Kit / AWS Kiro–style workflow for writing executable specs before code, with a constitution above the specs and a hard cross-check gate before implementation. **Who benefits:** Engineering teams using AI coding agents who keep getting "almost-right" implementations. **Why it matters:** an executable spec is the difference between an agent guessing your intent and an agent hitting the contract on the first try. **Why install now:** every PRD you write is wasted context until you have a spec format your agents treat as source of truth.

- **[`spec-driven-development`](.agents/skills/spec-driven-development/)** — Orchestrator. Routes `/constitution`, `/specify`, `/clarify`, `/plan`, `/tasks`, `/analyze`, `/implement` to the right leaf skill.
- **[`feature-spec`](.agents/skills/feature-spec/)** — Writes the executable feature spec (FRs, NFRs, ACs as Given/When/Then). Hard gate: cannot Approve while `[NEEDS CLARIFICATION]` markers remain.
- **[`project-constitution`](.agents/skills/project-constitution/)** — Authors `docs/constitution.md` — the non-negotiable engineering rules every spec, plan, and implementation must satisfy.
- **[`spec-crosscheck`](.agents/skills/spec-crosscheck/)** — Hard readiness gate before implementation. Cross-checks constitution + spec + plan + tasks for consistency, traceability, unresolved ambiguity. Returns PASS or FAIL.

### 💼 Venture Exploration Suite (5 skills) — *Pick the right business idea before you build it*

A pre-decision business-idea lifecycle — generate, model, evaluate, validate. Hands off to `product-soul` only after one idea passes a 5-criteria gate. **Who benefits:** founders, intrapreneurs, anyone deciding what to build next. **Why it matters:** the cheapest bug to fix is the one in the idea, not the code. **Why install now:** before your next "let's just build it" instinct fires.

- **[`venture-exploration`](.agents/skills/venture-exploration/)** — Orchestrator. Diagnoses lifecycle stage and routes to one child skill. Holds the binding 5/5 handoff gate.
- **[`idea-generation`](.agents/skills/idea-generation/)** — 5–10 structured idea cards using 8 generation methods. Hard-bans "Uber for X" / "AI for X" / "everyone" segments.
- **[`business-modeling`](.agents/skills/business-modeling/)** — Picks Lean Canvas / BMC / VPC for the stage and fills with falsifiable specifics. Bans generic channels and "unfair advantage = AI/data".
- **[`idea-evaluation`](.agents/skills/idea-evaluation/)** — Scores on 11 dimensions, returns binding GO / ITERATE / KILL with kill criteria + a next kill test.
- **[`customer-discovery`](.agents/skills/customer-discovery/)** — Mom Test–style interviews. Codes every quote into 7 categories with explicit verdict weights. Min 5 interviews before any positive verdict.

### 🎨 Frontend Design Suite (5 skills) — *Ship UI that doesn't look vibecoded*

The single biggest lever against AI-generated frontends: pick an archetype before any UI gets built. **Who benefits:** anyone shipping a frontend with AI tools. **Why it matters:** "Inter-on-purple-gradient with Lucide-everywhere and a centered hero + 2 CTAs" is the recognisable AI-generated-UI fingerprint that loses you trust. **Why install now:** the next UI you ship will look like every other AI-generated SaaS unless you fight back from the archetype up.

- **[`frontend-design`](.agents/skills/frontend-design/)** — Orchestrator. Routes through archetype → tokens → icons → build → review with anti-vibecoded hard gates.
- **[`design-archetype`](.agents/skills/design-archetype/)** — Picks one of 12 curated archetypes (Linear, Stripe, Apple, Duolingo, Posthog, Substack, Warp, ChatGPT, Leonardo, FigJam, Reddit, Vercel marketing).
- **[`design-tokens-craft`](.agents/skills/design-tokens-craft/)** — Archetype-driven semantic oklch tokens with hand-tuned dark mode. Bans Tailwind defaults and purple→pink gradients.
- **[`icon-craft`](.agents/skills/icon-craft/)** — 5 icon strategies that solve the "Lucide everywhere" tell.
- **[`design-review`](.agents/skills/design-review/)** — 10-dimension rubric review against the archetype's "feels like X" promise.

## Added — Workflow & Library Skills

- **[`generate-changelog` v1.1](.agents/skills/generate-changelog/)** — **Release discipline as a built-in workflow.** Adds the Four-Dimension Test (every entry answers WHAT / WHO / WHY MATTERS / WHY ACT NOW), Significance Triage gate (MAJOR / MINOR / PATCH), Accessibility & Motivation Check (changelog and README copy must hook + name outcome + give a CTA), and conditional README + release-push steps gated on significance. **Maintainers can now stop writing changelogs nobody reads.** Try it now if your last release notes felt like a commit dump.
- **[`problem-to-plan`](.agents/skills/problem-to-plan/)** — Tactical fast path. Turns a small problem or narrow refactor into three deliverables: a brief change-spec, a detailed implementation plan, and a TODO.md with agent-pickable tasks. **Use this** when the work is bug-sized but you still want agents to execute it cleanly.
- **[`learn-from`](.agents/skills/learn-from/)** + **[`learn-from-paper`](.agents/skills/learn-from-paper/)**, **[`learn-from-repo`](.agents/skills/learn-from-repo/)**, **[`learn-from-article`](.agents/skills/learn-from-article/)**, **[`learn-from-chat`](.agents/skills/learn-from-chat/)** — Auto-detects source type (paper / repo / article / in-conversation) and routes to the right ingestion skill with credibility check + security scan + insight extraction. **Use this** to pull validated insights from external sources into your skills or current project.
- **[`apply-paper-to-project`](.agents/skills/apply-paper-to-project/)** — Applies validated paper insights directly to the current codebase (architecture, patterns, testing, workflows). Always called after `learn-from-paper` has cleared the credibility and security gates.
- **[`cross-link-skills`](.agents/skills/cross-link-skills/)** — Repairs and verifies cross-references between SKILL.md files after creation, rename, removal, or restructure. Auto-called by `universal-skill-creator` and `improve-skills`.
- **`secure-skill` family expanded** — `secure-skill-content-sanitization`, `secure-skill-repo-ingestion`, `secure-skill-runtime` — three sub-skills covering hidden-content attacks, repo ingestion threats, and runtime state-corruption protection. Mandatory in every external-content workflow.

## Changed — Library Quality Hardening

- **New mandatory invariant in `AGENTS.md`** — Skill Creation Invariant. No agent may write SKILL.md directly; every creation routes through `universal-skill-creator` so the Step 8 quality chain (`validate-skills` → `skill-deconflict` → `library-skill`, optional `publish-skill`) fires automatically. **Stops** the most common library-rot pattern: agents bypassing the creator after a planning phase.
- **`universal-skill-creator` v2.1 → v2.2** — description triggers expanded to cover plural / suite / batch / go-ahead phrasings (`"build all skills"`, `"build the suite"`, `"go ahead build"`, `"ship the skills"`, etc.) so post-planning execution can no longer route past the creator. New Gotchas section (7 entries) codifies the hardest-won failure modes.
- **`compress-skill` Gotchas section added (6 entries)** — `secure-*` skills are uncompressable (180-line threshold; only allowed action is `split-skill`); description compression must be tested against ≥3 sample prompts; "move to references/" without a load trigger is invisible; preserve numbers, thresholds, and MUST/NEVER verbs; a 200-line skill that lost a hard gate is worse than a 220-line skill that kept it.
- **Experimentation suite (all 5 skills) — Gotchas baked in (33 total)** — covers the rigour failure modes most teams hit at least once: A/B-where-holdout-was-needed, MDE-relative-vs-absolute, exposure-event-at-flag-fetch, retrofitted "Causal" claims, ignored SRM failures, post-hoc significance theatre.
- **Library docs synced** — `README.md`, `docs/SKILL-INDEX.md`, `docs/skill-graph.md`, `docs/skill-outputs/SKILL-OUTPUTS.md`, `AGENTS.md` all reflect every new suite, new entry point, and new output path.

## Fixed

- **Stale cross-references after skill renames** — `cross-link-skills` is auto-called by `universal-skill-creator` and `improve-skills` so renames no longer leave dangling pointers in SKILL.md, AGENTS.md, README.md, or `docs/SKILL-INDEX.md`.
- **Changelogs no longer default to feature dumps** — three new gotchas in `generate-changelog` catch "lists changes without telling readers what to do", "internal jargon in user-facing copy", and "README What's New with no CTA".
- **Missing-from-README skill suites** — Spec-Driven Development (4 skills), Learn-From (5 skills), `problem-to-plan`, `apply-paper-to-project`, `cross-link-skills` are now indexed in README skill tables. Discoverability gap closed.
