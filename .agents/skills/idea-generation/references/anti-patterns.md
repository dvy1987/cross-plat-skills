# Idea Generation — Anti-Patterns

Load in `idea-generation` Step 4. Apply AFTER the batch is generated, not during. Filtering during generation kills quantity. Each anti-pattern: strike OR rewrite the card.

## Format anti-patterns (auto-strike unless rewritten)

- **"X for Y" labels** — "Uber for dog walkers", "Notion for lawyers", "AI for therapy". A label is not an idea. Force a concrete JTBD. If the user can't supply one, strike.
- **No segment** — "Consumers want…", "Developers need…", "Businesses struggle with…". No specific persona = no idea.
- **No current alternative** — "Nobody is solving this." Almost always wrong; the user's "nothing" is a manual workaround, an Excel sheet, or an ignored pain. If literally nothing → the pain isn't acute, strike.
- **No why-now** — A real why-now points to a specific shift in tech / regulation / cost / behaviour in the last 24 months. "AI" / "post-COVID" / "everything is online now" are not why-nows.

## Distribution anti-patterns (rewrite or strike)

- **Generic GTM** — "SEO + content + social" without naming the wedge keyword, sub-community, or list. Force one channel + specific entry point.
- **"Two-sided marketplace"** with no bootstrap-side — strike unless the founder names which side gets seeded first and how.
- **B2B without a named buyer** — for enterprise / mid-market ideas, the card must name role + budget owner + procurement path or it's an idea, not a venture.

## Viability anti-patterns (rewrite or flag)

- **"We'll figure out monetisation"** — only acceptable for genuinely consumer-distribution-first ideas (B2C-to-B2B with massive scale prerequisite). For SaaS / B2B / marketplace: rewrite with concrete model.
- **"10x better/faster/cheaper"** without a metric — force: faster from N to M, cheaper from $X to $Y.
- **TAM-first ideas** — "the freelance market is $1.5T" is not generative. Reverse: "this segment of 200k US freelancers each spending ~$X on tooling = $Y SOM".

## Defensibility anti-patterns (rewrite if user wants this idea)

- **"Unfair advantage = AI/data/network effects/community"** without a named asset — push for: proprietary data source, exclusive partnership, founder credibility, regulatory licence, or specific switching-cost mechanism.
- **"First-mover"** — almost never durable as a moat. Don't strike, but flag.

## ICP anti-patterns (strike)

- **Friend/family-only ICP** — "I asked my designer friend and she said yes". Friends/family are not validation; they are noise. Strike unless the friend is independently in the segment.
- **Solution-first framing** — "I want to build an app that uses [tech] to do [thing]". No pain identified = no idea. Reframe to pain-first or strike.

## Diversity anti-patterns (force regeneration)

- **>50% of cards cluster on one theme** — generate 2–3 more candidates from a different method.
- **All cards from one method** — pull from at least 2 methods (per Step 2).
- **All "obvious" candidates** — at least 2 cards must be tagged non-obvious. If none, push founder for schlep-blindness or live-in-the-future prompts.

## Strike log format

Every struck card is logged in the batch doc:
```
- <pitch> — struck — <which anti-pattern>
```

Logging matters: it teaches the user the filter so the next generation cycle is sharper.
