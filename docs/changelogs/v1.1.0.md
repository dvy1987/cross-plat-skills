# v1.1.0 — LLM Output Evaluation Suite & Learn-From Improvements

**Date:** 2026-04-19

This release adds a complete evaluation skill suite for scoring LLM and agent outputs — rubric design, LLM-as-judge scoring with bias mitigation, and automated eval pipeline design. It also improves the learn-from family with learnings provenance tracking, and hardens skill-deconflict to handle internal (caller-only) skills correctly.

---

## Breaking Changes

- None.

## Added

- **`eval-output`** — new orchestrator skill for evaluating LLM and agent outputs. Routes to the right sub-skill based on what the user needs: rubric design, direct scoring/pairwise comparison, or automated pipeline design. Enforces "no scoring without criteria" — always requires an explicit rubric.

- **`eval-rubric-design`** — design structured evaluation rubrics with quality dimensions, scoring scales (pass/fail for hard gates, 1-5 for quality), concrete score descriptions, and edge case guidance. Outputs rubric docs to `docs/evals/`. Supports both human reviewers and LLM judges.

- **`eval-judge`** — score outputs using LLM-as-judge techniques. Two modes: direct scoring against rubrics and pairwise comparison between two outputs. Built-in bias mitigation for position bias (mandatory position swap), length bias, self-enhancement bias, and verbosity bias. Justification-before-score pattern improves reliability 15-25%.

- **`eval-pipeline`** — design automated, multi-layer evaluation pipelines combining deterministic checks, statistical metrics, and LLM-as-judge scoring. Covers CI/CD integration, dataset design with four splits (happy path, edge cases, adversarial, known-bad), baseline establishment, alerting thresholds, and cost budgeting.

- **Learnings provenance tracking** — `learn-from` and sub-skills now track where learnings are stored (`docs/learnings/research-learnings.md`, `docs/learnings/chat-learnings.md`) and write back skill creation metadata to source learning entries.

## Changed

- **`skill-deconflict`** — now supports internal (caller-only) skills via `metadata.internal: true`. Internal skills are exempt from the 5-trigger minimum and diversity checks, but still checked for naming collisions and trigger overlaps.

- **`learn-from`** — updated learnings file paths from `docs/research-learnings/` to `docs/learnings/research-learnings.md`. Added learnings provenance requirement: newly created skills must be written back to the source learning entry.

- **`learn-from-article`** — improved with updated file paths and provenance tracking.

- **`learn-from-chat`** — improved with updated learnings storage path.

- **`universal-skill-creator`** — added learnings provenance tracking requirement for skills created from learnings files.

- **`skill-routing`** — added routing table entries for `eval-output` and `eval-rubric-design`.

- **AGENTS.md** — added eval-output entry points ("evaluate output", "score this response", "run an eval").

- **SKILL-INDEX.md** — added 4 eval-output family entries and updated call graph.
